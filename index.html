영국 전자담배세 인상과 소비자 선택

2025년 영란은행이 기준금리를 4.25%에서 4%로 내린 결정을 AD-AS 모형으로 뜯어봤습니다. 금리를 내리면 단기적으로는 GDP와 고용에 도움이 되지만, 물가가 목표치보다 높고 재정적자도 이미 큰 상황이라 이 정책을 얼마나 오래 끌고 갈 수 있을지는 의문이라는 결론을 냈습니다.

도시경제 데이터 분석: 동거·이혼 트렌드

IPUMS 인구총조사 마이크로데이터로 미국 18~30세 청년층의 동거율과 이혼율 사이에 어떤 관계가 있는지 살펴봤습니다.

  • 주(state)별로 동거율과 이혼율 차이를 지도로 그리고, 연도가 지나면서 어떻게 바뀌는지 GIF 애니메이션으로 만들었습니다.
  • 노동참여율이 높은 지역일수록, 동거가 이혼율을 낮추는 효과가 더 크게 나타났습니다.
  • 교육수준이 높으면 이혼율 자체는 낮아지지만, 반대로 동거가 주는 '보호 효과'는 약해지는 것으로 나타났습니다.
Python · NumPy

머신러닝 알고리즘 밑바닥부터 구현

라이브러리 없이 선형회귀, 로지스틱회귀, K-평균 군집화를 직접 코드로 짜봤습니다. 경사하강법이 손실함수를 줄여나가는 과정을 눈으로 보이게 시각화했습니다.

머신러닝 라이브러리를 쓰지 않고 세 가지 알고리즘을 처음부터 직접 짜보면서, 안에서 실제로 어떤 계산이 일어나는지 이해하려고 한 프로젝트입니다.

  • 선형회귀 — TV 광고비로 매출을 예측하는 모델을 그리드 서치, 경사하강법, 정규방정식(closed-form) 세 가지 방법으로 각각 만들어보고 결과를 비교했습니다. 손실함수가 줄어드는 과정은 2D/3D로 시각화했습니다.
  • 로지스틱회귀 — 리그 오브 레전드 경기 데이터로 골드 격차만 보고 승패를 예측하는 모델을 만들었습니다. 시그모이드 함수와 음의 로그우도 손실을 직접 구현했고, 결정경계가 학습하면서 점점 맞춰지는 과정을 애니메이션으로 만들었습니다.
  • K-평균 군집화 — 중심점을 정하고, 데이터를 가까운 군집에 다시 배정하고, 중심점을 갱신하는 과정을 반복 구현했습니다. 반복할 때마다 군집이 어떻게 바뀌는지 시각화했습니다.
PyTorch · GPU AI & 금융공학

딥헤징 (Deep Hedging with Neural Networks)

옵션을 헤지하는 전략을 신경망이 직접 학습하게 만들고, 전통적인 블랙숄즈 모형과 비교해봤습니다. 실시간 시세를 받아오는 파이프라인도 함께 구현했습니다.

정해진 공식(블랙숄즈)을 그대로 따르는 대신, 신경망이 데이터를 보고 스스로 헤징 전략을 찾아내도록 하는 '딥헤징' 방식을 구현했습니다.

  • 기하 브라운 운동으로 주가가 움직이는 경로를 여러 개 시뮬레이션했습니다.
  • 유러피언, 룩백, 아메리칸 바이너리, 유러피언 바이너리 등 옵션 종류별로 payoff 함수를 직접 짰습니다.
  • 신경망이 리스크(entropic risk measure)를 최소화하는 방향으로 헤징 전략을 학습하도록 PyTorch로 구현하고 GPU로 학습시켰습니다.
  • 학습된 전략을 블랙숄즈 공식, 몬테카를로 시뮬레이션 결과와 비교해서 잘 작동하는지 검증했습니다.
  • 여기서 그치지 않고 한국투자증권 Open API를 연동해 실시간 시세를 받아오는 파이프라인도 만들어, 이론이 실제 데이터에서도 통하는지 직접 연결해봤습니다.
Keras · TensorFlow

Fashion-MNIST 이미지 분류

옷 이미지를 분류하는 MLP 모델을 만들고, 하이퍼파라미터를 하나씩 조정해가며 테스트 정확도를 89.44%까지 끌어올렸습니다.

운동화, 셔츠, 가방 등 10가지 옷 이미지를 분류하는 Fashion-MNIST 데이터셋으로 3층짜리 MLP 모델을 만들었습니다.

  • Keras Sequential API로 은닉층 2개짜리 모델을 설계했습니다.
  • epoch, learning rate, batch size, hidden units을 하나씩 바꿔가며 어떤 조합이 제일 잘 맞는지 실험했습니다.
  • 최종 테스트 정확도 89.44%를 기록했습니다.
Python · NLTK · NetworkX

텍스트 그래프 마이닝 (TextRank)

구글 검색에 쓰이는 PageRank 알고리즘을 텍스트에 적용해서, 키워드를 뽑고 문서를 요약하는 걸 직접 구현했습니다.

구글이 웹페이지 순위를 매길 때 쓰는 PageRank 알고리즘을 문장과 단어에 적용한 게 TextRank입니다. 이 아이디어를 그래프 이론으로 직접 구현해봤습니다.

  • 키워드 추출 — 문장에서 같이 등장하는 단어끼리 연결한 그래프를 만들고, 여기에 PageRank를 적용해서 중요한 키워드를 뽑아냈습니다.
  • 추출 요약 — 문장끼리 얼마나 비슷한지(코사인 유사도)로 그래프를 만들고, 가장 중요한 문장만 골라내서 자동으로 요약했습니다.
